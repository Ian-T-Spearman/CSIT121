My experience

Creating my first website with HTML was an interesting, albeit intimidating, experience. I knew nothing of web development or what went on under the hood of websites, but after reading the module and looking at the external resources, it makes a lot more sense. Things are very logical, and structure is king. All of the content of a site can be likened to multiple matryoshka dolls standing next to each other, working in tandem. Working through the code and using the tricks that were taught in class, and also learned from the website, as well as using the validator to correct the many mistakes that I missed proved to be very satisfying work and quite enjoyable!

Things that I learned

  1. White space does not matter for HTML, which is similar to other markup languages, like XML.
  2. The difference between semantic elements and non-semantic elements: Semantic clearly defines its content, while Non-Semantic does not!
  3. You must define the language for your HTML document, and you must also make sure that you are using the correct character encoding. Otherwise the document will not know for sure how to correctly display the characters that you write, leading to unintended side effects (bugs).
  4. Attention to detail is very important! Sometimes things will not work and you won't know why, which happened to me. When I was creating the links for the websites that helped me, I accidentally had it written so that my copyright was included in the hyperlink and I couldn't figure out why. It wasn't until I looked through the line letter to letter that I saw that I forgot to close the "< a href="" < /a> >" tag.
  5. Footer tags must never be placed after the body tag!
